## Hook вҖ” Bits on a Wire

Everything starts with physics. A voltage change on a copper wire. A pulse of light in a fiber optic cable. A radio wave from an antenna. The **physical layer** turns bits into signals, and the **data link layer** ensures those bits arrive correctly across a single hop.

### Error Detection

```
CRC (Cyclic Redundancy Check):
  Treat data as polynomial, divide by generator.
  Remainder = checksum. Append to frame.
  Receiver divides again вҖ” if remainder вү  0, error detected.
  
  Detects: all single-bit errors, all double-bit errors,
           any odd number of errors, all burst errors < checksum length.

  CRC-32: 99.99999% detection rate for random errors.
```

### Multiple Access вҖ” Sharing the Wire

```
How do multiple devices share one communication channel?

ALOHA: Send whenever ready. Collision? Wait random time, retry.
  Efficiency: ~18% (pure), ~37% (slotted).

CSMA/CD (Ethernet):
  1. Listen before sending (Carrier Sense)
  2. If busy, wait. If clear, send.
  3. Listen while sending (Collision Detection)
  4. If collision: stop, send jam signal, wait random time (exponential backoff)
  
  Not needed for full-duplex switched Ethernet (modern standard).

CSMA/CA (WiFi):
  Can't detect collisions wirelessly (half-duplex).
  1. Listen. If clear, wait DIFS + random backoff.
  2. Send. Wait for ACK from receiver.
  3. No ACK? Collision assumed вҶ’ increase backoff, retry.
```

## Why It Matters

- Understanding L1/L2 explains why WiFi is unreliable (collisions, interference) and Ethernet is reliable.
- CRC is used everywhere вҖ” Ethernet, USB, ZIP files, PNG images вҖ” for error detection.
- MTU understanding prevents fragmentation problems and explains "path MTU discovery."
