## Hook — Building Networked Applications

Every networked application — web servers, chat apps, multiplayer games — uses **sockets**. A socket is an endpoint for communication: an IP address + port number. Understanding sockets is understanding how programs talk over the network.

## What Is a Socket?

```
Socket = (IP address, Port number, Protocol)

Example: (192.168.1.100, 8080, TCP)

Well-known ports (0-1023):
  80   HTTP
  443  HTTPS
  22   SSH
  53   DNS
  25   SMTP

Ephemeral ports (49152-65535):
  Assigned by OS for client-side connections.
  
  Client: 192.168.1.100:52431 → Server: 93.184.216.34:443
```

## See It Work — TCP Server/Client in C

### Server

```c
// 1. Create socket
int server_fd = socket(AF_INET, SOCK_STREAM, 0);

// 2. Bind to address
struct sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_addr.s_addr = INADDR_ANY;  // listen on all interfaces
addr.sin_port = htons(8080);         // port 8080
bind(server_fd, (struct sockaddr*)&addr, sizeof(addr));

// 3. Listen for connections
listen(server_fd, 10);  // backlog queue of 10

// 4. Accept connection (blocks until client connects)
int client_fd = accept(server_fd, NULL, NULL);

// 5. Read/Write
char buf[1024];
int n = read(client_fd, buf, sizeof(buf));
write(client_fd, "Hello!\n", 7);

// 6. Close
close(client_fd);
close(server_fd);
```

### Client

```c
int sock = socket(AF_INET, SOCK_STREAM, 0);

struct sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_port = htons(8080);
inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);

connect(sock, (struct sockaddr*)&addr, sizeof(addr));
write(sock, "Hi server!", 10);

char buf[1024];
int n = read(sock, buf, sizeof(buf));
printf("Server said: %s\n", buf);
close(sock);
```

## How It Actually Works — Handling Multiple Clients

```
Problem: accept() and read() block.
  Server handles one client at a time!

Solution 1: Fork per connection
  accept() → fork() → child handles client.
  Simple but expensive (process per connection).

Solution 2: Thread per connection
  accept() → pthread_create() → thread handles client.
  Better but still limited (~10K threads max).

Solution 3: Event-driven I/O (select/poll/epoll)
  Single thread monitors MANY sockets.
  epoll (Linux): O(1) notification when any socket is ready.
  
  This is how nginx, Node.js, Redis work.
  Can handle 100K+ concurrent connections on one thread!

  // Pseudocode:
  epoll_fd = epoll_create();
  epoll_ctl(epoll_fd, ADD, server_fd);  // watch server socket
  while (true) \{
    events = epoll_wait(epoll_fd);  // blocks until activity
    for each ready socket:
      if server_fd: accept() new client, add to epoll
      if client_fd: read/write data
  \}
```

### UDP Sockets

```c
// No connect/accept — just send/receive
int sock = socket(AF_INET, SOCK_DGRAM, 0);
bind(sock, ...);

// Receive from anyone
recvfrom(sock, buf, sizeof(buf), 0, &sender_addr, &len);

// Send to specific address
sendto(sock, "response", 8, 0, &sender_addr, len);

// No connection state, no ordering, no reliability
// Application must handle these if needed
```

## Why It Matters

- Every web server, database client, and chat application is built on sockets.
- Understanding blocking vs non-blocking I/O explains why some servers handle millions of connections.
- Socket programming is the foundation for understanding distributed systems.
