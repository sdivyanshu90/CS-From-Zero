## Hook — Finding the Path

Your packet needs to reach a server 10,000 km away, crossing dozens of networks. No single router knows the full path. Each router makes a local decision: "which neighbor should I forward this to?" **Routing protocols** are how routers learn these paths and adapt when links fail.

## How Routing Works

```
Routing Table (simplified):
Destination     Next Hop        Interface   Metric
10.0.0.0/8      192.168.1.1     eth0        1
172.16.0.0/16   192.168.1.254   eth0        3
0.0.0.0/0       192.168.1.1     eth0        1  ← default route

Longest Prefix Match:
  Packet to 10.1.2.3 → matches 10.0.0.0/8 → next hop 192.168.1.1
  Packet to 172.16.5.1 → matches 172.16.0.0/16 → next hop 192.168.1.254
  Packet to 8.8.8.8 → no specific match → default route
```

## See It Work — Routing Algorithms

### Distance Vector (RIP)

```
Each router tells neighbors its distances to all destinations.
"I can reach network X in 3 hops."

Bellman-Ford algorithm:
  Dx(y) = min over neighbors v [ cost(x,v) + Dv(y) ]

Problem: Count-to-infinity
  Link fails → routers slowly increment distance.
  A thinks B can reach X in 3 hops.
  B thinks A can reach X in 3 hops.
  Both increment forever!
  
  Fix: Split horizon (don't advertise route back to sender).
  Fix: Poison reverse (advertise ∞ back to sender).

RIP: Max 15 hops. Updates every 30 seconds. Rarely used today.
```

### Link State (OSPF)

```
Each router floods information about its DIRECTLY connected links.
Every router builds a complete map of the network.
Then runs Dijkstra's algorithm to find shortest paths.

Advantages over Distance Vector:
  - Faster convergence (no count-to-infinity)
  - Each router has complete topology
  - Scales better (hierarchical areas)

OSPF: Open Shortest Path First.
  Used inside organizations (Intra-domain / IGP).
  Supports areas for hierarchy.
```

### Path Vector (BGP)

```
BGP: Border Gateway Protocol — the internet's routing protocol.
Routes between Autonomous Systems (ISPs, organizations).

Instead of distance, BGP advertises the FULL PATH:
  "To reach 8.8.8.0/24, go through AS15169 → AS3356 → AS7018"

This prevents loops (if your AS is in the path, reject it).

BGP is policy-based:
  ISPs choose routes based on business relationships,
  not just shortest path.
  
  Customer → Provider: customer pays for transit
  Peer → Peer: free exchange (settlement-free peering)
  
~900K routes in the global BGP table (2024).
```

## How It Actually Works — traceroute

```bash
$ traceroute google.com
 1  192.168.1.1      1ms     ← home router
 2  10.0.0.1         5ms     ← ISP router
 3  72.14.194.1      15ms    ← backbone router
 4  142.250.190.14   20ms    ← Google's network

How it works: Send packets with increasing TTL.
  TTL=1 → first router decrements to 0, sends ICMP "Time Exceeded"
  TTL=2 → second router responds
  TTL=3 → third router responds
  ... until destination reached
```

## Why It Matters

- Routing is what makes the internet work — without it, packets can't cross network boundaries.
- BGP misconfiguration or hijacking can take down major portions of the internet (has happened!).
- Understanding routing helps debug connectivity issues and design network architectures.
