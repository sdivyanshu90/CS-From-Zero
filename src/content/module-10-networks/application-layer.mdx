## Hook — Protocols You Use Every Day

Every time you browse the web, send email, or stream video, application layer protocols make it happen. **HTTP** fetches web pages, **DNS** translates names to addresses, **SMTP** delivers email. These are the protocols that users interact with, built on top of TCP and UDP.

## DNS — The Internet's Phone Book

```
You type: www.google.com
Browser needs: 142.250.190.14

DNS Resolution (iterative):
  1. Browser → Local DNS resolver: "Who is www.google.com?"
  2. Resolver → Root server (.): "Who handles .com?"
  3. Resolver → .com TLD server: "Who handles google.com?"
  4. Resolver → google.com authoritative: "www = 142.250.190.14"
  5. Resolver caches result (TTL = 300 seconds)
  6. Returns IP to browser.

Record Types:
  A      IPv4 address       google.com → 142.250.190.14
  AAAA   IPv6 address       google.com → 2607:f8b0::200e
  CNAME  Alias              www.google.com → google.com
  MX     Mail server        google.com → smtp.google.com
  NS     Name server        google.com → ns1.google.com
  TXT    Arbitrary text      SPF, DKIM (email auth)
```

## HTTP — The Web's Protocol

```
Request:
  GET /index.html HTTP/1.1
  Host: www.example.com
  User-Agent: Mozilla/5.0
  Accept: text/html

Response:
  HTTP/1.1 200 OK
  Content-Type: text/html
  Content-Length: 1234
  
  <html>...</html>

Methods: GET (read), POST (create), PUT (update), DELETE (remove)
Status: 200 OK, 301 Redirect, 404 Not Found, 500 Server Error
```

### HTTP Evolution

```
HTTP/1.0: One request per TCP connection. Slow.
HTTP/1.1: Keep-alive (reuse connection). Pipelining. Still head-of-line blocking.
HTTP/2:   Multiplexing (multiple requests over one connection).
          Binary framing. Header compression. Server push.
HTTP/3:   Built on QUIC (UDP-based). No TCP head-of-line blocking.
          Faster connection setup (0-RTT). Built-in encryption.
```

## See It Work — HTTPS & TLS

```
TLS (Transport Layer Security) = encryption for HTTP.

TLS Handshake:
  1. Client Hello: supported ciphers, random number
  2. Server Hello: chosen cipher, certificate (public key)
  3. Client verifies certificate (trusted CA?)
  4. Key exchange (Diffie-Hellman): derive shared secret
  5. Encrypted communication begins

Certificate Chain:
  Server cert → signed by Intermediate CA → signed by Root CA
  Root CAs are trusted by your OS/browser (~100 root CAs).

TLS 1.3: 1-RTT handshake (vs 2-RTT in TLS 1.2). Faster + more secure.
```

### Other Application Protocols

```
SMTP: Send email (port 25/587)
IMAP: Read email (port 993)
SSH:  Secure shell (port 22)
FTP:  File transfer (port 21) — mostly replaced by SFTP/SCP
DHCP: Automatic IP assignment (port 67/68)
NTP:  Time synchronization (port 123)
```

## Why It Matters

- HTTP is the backbone of the web — understanding it is essential for any web developer.
- DNS is a critical piece of internet infrastructure — DNS failures take down entire services.
- TLS/HTTPS is mandatory for security — understanding it prevents misconfigurations.
