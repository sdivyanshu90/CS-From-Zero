## Hook — How Does This Page Reach You?

You type a URL. Milliseconds later, a page appears from a server thousands of miles away. Between your click and the response, data travels through cables, routers, switches, and protocols — a global communication system more complex than anything humans have ever built. **Computer networking** is how it all works.

## The OSI Model — Seven Layers

```
Layer  Name           What It Does               Example
─────  ────           ────────────               ───────
7      Application    User-facing protocols       HTTP, DNS, SMTP
6      Presentation   Data formatting/encryption  TLS, JPEG, JSON
5      Session        Connection management       Sessions, auth
4      Transport      Reliable delivery           TCP, UDP
3      Network        Routing across networks     IP, ICMP
2      Data Link      Hop-to-hop delivery         Ethernet, WiFi
1      Physical       Bits on the wire            Cables, radio
```

### TCP/IP Model (What's Actually Used)

```
TCP/IP Layer    ↔  OSI Layers    Protocols
────────────       ──────────    ─────────
Application        5, 6, 7      HTTP, DNS, SMTP, SSH
Transport          4             TCP, UDP
Internet           3             IP, ICMP, ARP
Link               1, 2         Ethernet, WiFi, PPP
```

## See It Work — A Packet's Journey

```
You type: https://example.com/page

1. DNS: "What's the IP for example.com?" → 93.184.216.34
2. TCP: Three-way handshake (SYN → SYN-ACK → ACK)
3. TLS: Negotiate encryption (keys, certificates)
4. HTTP: GET /page HTTP/1.1
5. Server processes request, sends response
6. Data travels through routers across the internet
7. Browser renders the HTML

Each layer adds a header (encapsulation):
[Ethernet HDR | IP HDR | TCP HDR | HTTP Data | Ethernet Trailer]
     L2           L3       L4         L7            L2
```

## How It Actually Works — Encapsulation

```
Application:  "Hello World"
                    ↓ add HTTP header
Transport:    [TCP HDR | "Hello World"]
                    ↓ add IP header
Network:      [IP HDR | TCP HDR | "Hello World"]
                    ↓ add Ethernet header + trailer
Link:         [ETH HDR | IP | TCP | "Hello World" | ETH FCS]

At each router: strip L2 header, check L3 (IP), 
                add new L2 header for next hop.

At destination: strip headers layer by layer (decapsulation).
```

### Circuit Switching vs Packet Switching

```
Circuit Switching (old phone network):
  Dedicated path for entire call duration.
  Guaranteed bandwidth but wasteful (path reserved even during silence).

Packet Switching (the internet):
  Data split into packets. Each routed independently.
  Shared links. No reservation. Best effort.
  + Efficient: links shared by many users
  + Resilient: packets can take different paths
  - No guaranteed delivery or timing
```

## Why It Matters

- The layered model lets engineers work on one layer without understanding others.
- Understanding encapsulation is essential for debugging network issues (Wireshark, tcpdump).
- Every web developer, sysadmin, and security engineer needs networking fundamentals.
