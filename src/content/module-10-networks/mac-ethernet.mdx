## Hook — Your Device's Hardware Identity

Every network interface has a unique 48-bit **MAC address** burned into its hardware. While IP addresses change as you move between networks, your MAC address stays the same. It's how devices find each other on a local network — the "name tag" of the data link layer.

## MAC Addresses

```
Format: 6 bytes, written as hex pairs
  AA:BB:CC:DD:EE:FF

First 3 bytes (OUI): Manufacturer ID
  00:1A:2B = manufacturer X
  
Last 3 bytes: Device serial number

Special addresses:
  FF:FF:FF:FF:FF:FF = Broadcast (all devices on LAN)
  01:xx:xx:xx:xx:xx = Multicast (bit 0 of first byte = 1)
```

## See It Work — Ethernet Switching

```
Switch has a MAC Address Table (CAM table):
  Port 1 → AA:BB:CC:11:22:33
  Port 2 → AA:BB:CC:44:55:66
  Port 3 → AA:BB:CC:77:88:99

Frame arrives on Port 1, dest = AA:BB:CC:44:55:66:
  → Look up MAC table → Port 2 → forward only to Port 2.

Unknown destination?
  → Flood: send to ALL ports except source port.
  → When destination replies, learn its port.

This is why switches are "self-learning."
```

### ARP — Bridging IP and MAC

```
Problem: IP knows destination IP, but Ethernet needs MAC address.
Solution: Address Resolution Protocol (ARP).

Host A wants to send to 192.168.1.5:
  1. A broadcasts: "Who has 192.168.1.5? Tell 192.168.1.1"
     (Dest MAC: FF:FF:FF:FF:FF:FF)
  2. Host at 192.168.1.5 replies: "I'm at AA:BB:CC:DD:EE:FF"
  3. A caches this in ARP table (timeout ~20 min)
  4. Future packets use cached MAC directly.

$ arp -a   ← view ARP cache
$ ip neigh ← Linux equivalent
```

## How It Actually Works — VLANs

```
VLAN (Virtual LAN): Logically separate one switch into multiple LANs.

Physical switch, 24 ports:
  Ports 1-12: VLAN 10 (Engineering)
  Ports 13-24: VLAN 20 (Marketing)

Traffic between VLANs must go through a router (L3).
Even though devices are on the same physical switch,
they can't see each other across VLANs.

802.1Q: Adds 4-byte VLAN tag to Ethernet frame.
  [Dest MAC | Src MAC | 802.1Q Tag | Type | Data | FCS]
                        └─ VLAN ID (12 bits = 4096 VLANs)
```

## Why It Matters

- MAC addresses are how local network delivery works — every frame on your LAN uses them.
- Understanding ARP explains ARP spoofing attacks (man-in-the-middle on LANs).
- VLANs are the foundation of network segmentation in every office and data center.
