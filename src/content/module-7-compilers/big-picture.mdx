## Hook — From Text to Machine Code

You type `printf("Hello")` and the CPU executes binary instructions. How? A **compiler** is a program that translates source code (human-readable) into machine code (CPU-executable). It's one of the most sophisticated programs ever built — and understanding it makes you a dramatically better programmer.

## What Is a Compiler?

```
Source Code → [COMPILER] → Machine Code
  (C, Java)                  (x86, ARM)

Phases:
┌────────────────────────────────────────────────┐
│              Front End                          │
│  ┌──────────┐  ┌─────────┐  ┌───────────────┐ │
│  │ Lexical  │→│ Syntax  │→│  Semantic      │ │
│  │ Analysis │  │ Analysis│  │  Analysis      │ │
│  └──────────┘  └─────────┘  └───────────────┘ │
├────────────────────────────────────────────────┤
│              Middle                             │
│  ┌───────────────────┐  ┌──────────────────┐  │
│  │ Intermediate Code │→│  Optimization    │  │
│  └───────────────────┘  └──────────────────┘  │
├────────────────────────────────────────────────┤
│              Back End                           │
│  ┌───────────────────┐  ┌──────────────────┐  │
│  │ Code Generation   │→│  Machine Code    │  │
│  └───────────────────┘  └──────────────────┘  │
└────────────────────────────────────────────────┘
```

## See It Work — Compiling `x = a + b * c`

```
1. Lexical Analysis (Lexer):
   "x = a + b * c" → [ID:x] [ASSIGN] [ID:a] [PLUS] [ID:b] [STAR] [ID:c]

2. Syntax Analysis (Parser):
   Builds parse tree:
         =
        / \
       x   +
          / \
         a   *
            / \
           b   c

3. Semantic Analysis:
   Type checking: are a, b, c compatible types?
   Symbol table: are all variables declared?

4. Intermediate Code:
   t1 = b * c
   t2 = a + t1
   x = t2

5. Optimization:
   (Maybe constant folding, dead code elimination)

6. Code Generation:
   MOV R1, b      ; load b
   MUL R1, c      ; R1 = b * c
   ADD R1, a      ; R1 = a + b*c
   MOV x, R1      ; store result
```

## How It Actually Works — Compiler vs. Interpreter

```
Compiler:
  Source → Machine code → Execute (later)
  C, C++, Rust, Go
  + Fast execution
  - Compile time, platform-specific

Interpreter:
  Source → Execute (line by line, immediately)
  Python, Ruby, JavaScript (initially)
  + Immediate feedback, portable
  - Slower execution

Hybrid (JIT):
  Source → Bytecode → JIT compile hot paths → Execute
  Java (JVM), C# (.NET), JavaScript (V8)
  + Portable + fast for hot code
  - Warm-up time
```

### Other Translation Tools

```
Preprocessor:  Source → Modified Source    (#include, #define in C)
Assembler:     Assembly → Machine Code    (1-to-1 translation)
Linker:        Object files → Executable  (resolves external refs)
Transpiler:    Source → Source             (TypeScript → JavaScript)
```

## Why It Matters

- Understanding compilation helps you write faster code, debug better, and understand error messages.
- Compiler techniques appear everywhere: IDEs (syntax highlighting), linters, query engines, DSLs.
- The front-end/middle/back-end separation allows compiling M languages to N architectures with M+N components instead of M×N.
