## Hook — Connect Everything at Minimum Cost

Suppose you need to connect 10 cities with roads, minimizing total road length. You need a tree (connected, no cycles) that spans all cities with minimum total edge weight. That's a **Minimum Spanning Tree (MST)** — used in network design, clustering, and circuit layout.

## What Is an MST?

A **spanning tree** of graph G is a subgraph that:
- Includes all vertices
- Is a tree (connected, no cycles)
- Has exactly V-1 edges

The **minimum** spanning tree has the smallest total edge weight.

```
Graph:                    MST (total weight = 10):
A --3-- B                 A --3-- B
|     / |                         |
4   2   5                    2    
|  /    |                   /     
C --1-- D                 C --1-- D
                          |
                          4 ← wait, this doesn't work!
                          Actually: A-B(3), B-C(2), C-D(1) = 6
```

## See It Work — Two Classic Algorithms

### Kruskal's Algorithm — O(E log E)

Greedy: Sort edges by weight, add cheapest edge that doesn't create a cycle.

```
Edges sorted: (C,D,1) (B,C,2) (A,B,3) (A,C,4) (B,D,5)

Add (C,D,1): No cycle ✓  MST: {C-D}
Add (B,C,2): No cycle ✓  MST: {C-D, B-C}
Add (A,B,3): No cycle ✓  MST: {C-D, B-C, A-B}
3 edges = V-1 → done! Total weight = 6

Skip (A,C,4): Creates cycle A-B-C-A
Skip (B,D,5): Creates cycle B-C-D-B
```

Uses **Union-Find** (disjoint set) to efficiently detect cycles:
```c
// Nearly O(1) amortized operations with path compression + union by rank
int find(int parent[], int x) {
    if (parent[x] != x) parent[x] = find(parent, parent[x]);
    return parent[x];
}
void unite(int parent[], int rank[], int x, int y) {
    int px = find(parent, x), py = find(parent, y);
    if (rank[px] < rank[py]) parent[px] = py;
    else if (rank[px] > rank[py]) parent[py] = px;
    else { parent[py] = px; rank[px]++; }
}
```

### Prim's Algorithm — O((V + E) log V)

Greedy: Start from any vertex, always add the cheapest edge connecting the tree to a non-tree vertex.

```
Start at A:
  Cheapest edge from {A}: A-B(3). Add B.
  Cheapest edge from {A,B}: B-C(2). Add C.
  Cheapest edge from {A,B,C}: C-D(1). Add D.
  
MST: {A-B(3), B-C(2), C-D(1)} = 6. Same result!
```

Uses a priority queue, similar to Dijkstra's algorithm.

## How It Actually Works — Comparison

```
Algorithm   Time              Best for         Cycle detection
─────────   ────              ────────         ───────────────
Kruskal's   O(E log E)        Sparse graphs    Union-Find
Prim's      O((V+E) log V)    Dense graphs     Priority queue
```

Both produce the same MST (if edge weights are unique). If weights aren't unique, multiple valid MSTs may exist.

### Cut Property

The **cut property** proves both algorithms correct: for any cut (partition of vertices into two groups), the lightest edge crossing the cut must be in the MST.

## Why It Matters

- Network design: connecting computers, cities, or circuits at minimum cost.
- Clustering: removing the most expensive MST edges creates clusters.
- Approximation algorithms for NP-hard problems (e.g., traveling salesman).
