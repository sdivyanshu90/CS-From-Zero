## Hook — Finding the Best Route

Every time you use Google Maps, a shortest path algorithm runs. Given a graph with weighted edges (distances, times, costs), find the cheapest path from one vertex to another. This is one of the most practically important problems in CS.

## Dijkstra's Algorithm — O((V + E) log V)

For graphs with **non-negative edge weights**. Uses a greedy approach with a priority queue:

```
Graph:
A --1-- B --2-- D
|       |       |
4       1       3
|       |       |
C --5---+---1-- E

Dijkstra from A:
  dist: A=0, B=∞, C=∞, D=∞, E=∞

  Visit A (dist=0): Update B=1, C=4
  Visit B (dist=1): Update D=3, C=2 (1+1<4), E=∞
  Visit C (dist=2): Update (nothing improves)
  Visit D (dist=3): Update E=6
  Visit E (dist=4 via B→E=1+1+1... depends on edges)
  
Key idea: Always process the unvisited vertex with smallest distance.
          Once a vertex is processed, its distance is final.
```

```c
// Pseudocode
void dijkstra(Graph G, int source) {
    dist[source] = 0;  // all others = ∞
    priority_queue pq;
    pq.insert(source, 0);
    
    while (!pq.empty()) {
        u = pq.extract_min();
        for each neighbor v of u:
            if (dist[u] + weight(u,v) < dist[v]) {
                dist[v] = dist[u] + weight(u,v);
                prev[v] = u;
                pq.decrease_key(v, dist[v]);
            }
    }
}
// Time: O((V + E) log V) with binary heap
//       O(V² + E) with simple array (better for dense graphs)
```

**Dijkstra FAILS with negative edges** because it assumes processed vertices have final distances. A negative edge could provide a shortcut to an already-processed vertex.

## Bellman-Ford — O(V × E)

Handles **negative edge weights** and detects **negative cycles**:

```c
void bellman_ford(int V, int E, Edge edges[], int source) {
    int dist[V];
    fill(dist, INF);
    dist[source] = 0;
    
    // Relax all edges V-1 times
    for (int i = 0; i < V - 1; i++)
        for (int j = 0; j < E; j++)
            if (dist[edges[j].u] + edges[j].w < dist[edges[j].v])
                dist[edges[j].v] = dist[edges[j].u] + edges[j].w;
    
    // Check for negative cycles (one more pass)
    for (int j = 0; j < E; j++)
        if (dist[edges[j].u] + edges[j].w < dist[edges[j].v])
            printf("Negative cycle detected!\n");
}
```

**Why V-1 iterations?** The shortest path has at most V-1 edges. Each iteration correctly computes paths using one more edge.

## Floyd-Warshall — O(V³)

Finds shortest paths between **ALL pairs** of vertices:

```c
// dp[i][j] = shortest distance from i to j
for (int k = 0; k < V; k++)      // try each vertex as intermediate
    for (int i = 0; i < V; i++)
        for (int j = 0; j < V; j++)
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j]);
```

Beautifully simple. O(V³) but handles negative edges. Negative cycle exists if `dp[i][i] < 0`.

## How It Actually Works — Comparison

```
Algorithm        Edges      All-pairs?  Time           Negative edges
─────────        ─────      ──────────  ────           ──────────────
BFS              Unweighted Single-src  O(V + E)       N/A
Dijkstra         Non-neg    Single-src  O((V+E)log V)  No!
Bellman-Ford     Any        Single-src  O(VE)          Yes + detection
Floyd-Warshall   Any        All-pairs   O(V³)          Yes + detection
```

## Why It Matters

- GPS navigation, network routing (OSPF uses Dijkstra), and logistics all use shortest paths.
- Understanding which algorithm to use depends on edge weights and whether you need single-source or all-pairs.
