## Hook — Exploring Every Corner

How does Google Maps find a route? How does a web crawler visit every page? How does a social network suggest "people you may know"? All these start with **graph traversal** — systematically visiting every vertex and edge. BFS and DFS are the two fundamental traversal strategies, and nearly every graph algorithm builds on them.

## BFS — Breadth-First Search

Explore all neighbors before going deeper. Uses a **queue** (FIFO):

```c
void bfs(int graph[][MAX], int n, int start) {
    int visited[MAX] = {0};
    int queue[MAX], front = 0, rear = 0;
    
    visited[start] = 1;
    queue[rear++] = start;
    
    while (front < rear) {
        int v = queue[front++];
        printf("%d ", v);
        
        for (int u = 0; u < n; u++) {
            if (graph[v][u] && !visited[u]) {
                visited[u] = 1;
                queue[rear++] = u;
            }
        }
    }
}
```

```
        A
       / \
      B   C
     / \   \
    D   E   F

BFS from A: A → B, C → D, E, F (level by level)
Queue trace: [A] → [B,C] → [C,D,E] → [D,E,F] → [E,F] → [F] → []
```

### BFS Properties

- **Shortest path** in unweighted graphs (fewest edges)
- **Time**: O(V + E) — visits every vertex and edge once
- **Space**: O(V) — queue can hold up to V vertices
- Explores in "expanding rings" from the source

### BFS Applications

```
Application                What BFS gives you
───────────                ──────────────────
Shortest path (unweighted) Number of edges from source to every vertex
Connected components       Which vertices are reachable from each other
Bipartite check           Can the graph be 2-colored?
Level-order tree traversal Visit nodes level by level
Web crawling              Discover pages by following links
```

## DFS — Depth-First Search

Go as deep as possible before backtracking. Uses a **stack** (or recursion):

```c
void dfs(int graph[][MAX], int n, int v, int visited[]) {
    visited[v] = 1;
    printf("%d ", v);
    
    for (int u = 0; u < n; u++) {
        if (graph[v][u] && !visited[u]) {
            dfs(graph, n, u, visited);
        }
    }
}
```

```
        A
       / \
      B   C
     / \   \
    D   E   F

DFS from A: A → B → D → E → C → F (go deep, then backtrack)
```

### DFS Properties

- **Time**: O(V + E)
- **Space**: O(V) — recursion stack depth
- Discovers back edges (cycles) naturally

### DFS Applications

```
Application              How DFS helps
───────────              ─────────────
Cycle detection          Back edge in DFS tree = cycle
Topological sort         Process nodes in reverse finish order
Connected components     Each DFS tree is a component
Strongly connected       Tarjan's or Kosaraju's algorithm
  components (directed)
Path finding             Does a path exist from A to B?
Maze solving             Explore one path completely, backtrack
```

### Topological Sort via DFS

```
Process nodes in reverse DFS finish order:

void topo_dfs(int v, int visited[], int stack[], int *top) {
    visited[v] = 1;
    for each neighbor u of v:
        if (!visited[u]) topo_dfs(u, visited, stack, top);
    stack[(*top)++] = v;  // push to stack after all descendants done
}

Reverse the stack → topological order
```

## How It Actually Works — BFS vs. DFS Summary

```
Property          BFS                 DFS
────────          ───                 ───
Data structure    Queue               Stack (recursion)
Order             Level by level      As deep as possible
Shortest path     Yes (unweighted)    No
Cycle detection   Possible            Natural (back edges)
Space             O(V) — can be wide  O(V) — can be deep
Topological sort  Kahn's algorithm    Reverse finish order
```

## Why It Matters

- BFS and DFS are the building blocks of virtually all graph algorithms.
- BFS finds shortest paths (Dijkstra generalizes BFS for weighted graphs).
- DFS finds cycles, topological orderings, and strongly connected components.
- Understanding when to use BFS vs. DFS is a fundamental algorithmic skill.
