## Hook — Finding Needles in Haystacks

You search for things constantly: a file on your computer, a word in a document, a contact in your phone, a product on Amazon. The difference between a good search and a bad one is the difference between instant results and waiting minutes. Search algorithms are so fundamental that entire companies (Google) are built around them.

## Linear Search — O(n)

Check each element one by one:

```c
int linear_search(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) return i;
    }
    return -1;
}
// Best: O(1), Worst: O(n), Average: O(n/2) = O(n)
// Works on unsorted data. Simple but slow for large arrays.
```

## Binary Search — O(log n)

**Requires sorted data.** Repeatedly halve the search space:

```c
int binary_search(int arr[], int n, int target) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;  // avoids overflow
        if (arr[mid] == target) return mid;
        if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}
```

```
Search for 7 in [1, 3, 5, 7, 9, 11, 13, 15]:
Step 1: mid=3 (value 7). Found! → return 3

Search for 11:
Step 1: mid=3 (7). 11 > 7 → search [9,11,13,15]
Step 2: mid=5 (11). Found! → return 5

Each step eliminates half → O(log n)
1 billion elements: only ~30 comparisons!
```

### Binary Search Variants

```
Lower bound: First element ≥ target
Upper bound: First element > target
First occurrence: Binary search + check left neighbor
Last occurrence: Binary search + check right neighbor

All O(log n). Used extensively in competitive programming.
```

## How It Actually Works — Search in Different Data Structures

```
Data Structure        Search Time    Algorithm
──────────            ───────────    ─────────
Unsorted array        O(n)           Linear search
Sorted array          O(log n)       Binary search
Hash table            O(1) avg       Hash function
BST (balanced)        O(log n)       Tree traversal
BST (unbalanced)      O(n)           Tree traversal
Trie                  O(m)           Prefix matching (m = key length)
```

### Interpolation Search

Like binary search but guesses where the target might be based on value distribution (like how you'd search a phone book — names starting with Z are near the end):

```
For uniformly distributed data: O(log log n) average
For non-uniform data: O(n) worst case
```

### Ternary Search

Divides into three parts instead of two. Useful for finding the minimum/maximum of a unimodal function (not for sorted arrays — binary search is faster).

## Why It Matters

- Binary search is the most important O(log n) algorithm — used in databases, file systems, and libraries.
- The "sorted data" requirement connects searching to sorting — often you sort once, search many times.
- Understanding search complexity helps you choose the right data structure.
