## Hook — The Ultimate Computer

In 1936, Alan Turing imagined a simple machine: an infinite tape, a read/write head, and a finite set of rules. This thought experiment defined what it means to "compute" and remains the foundation of all computer science. Every algorithm ever written, every program ever executed, can be simulated by a Turing machine.

## What Is a Turing Machine?

A TM is a 7-tuple (Q, Σ, Γ, δ, q₀, q_accept, q_reject):

```
Q = finite set of states
Σ = input alphabet (no blank symbol)
Γ = tape alphabet (Σ ∪ \{blank\})
δ = transition function: Q × Γ → Q × Γ × \{L, R\}
q₀ = start state
q_accept = accept state
q_reject = reject state
```

### Example: Recognize \{0ⁿ1ⁿ | n ≥ 1\}

```
Strategy: Cross off one 0 and one 1 repeatedly.

Tape: [0 0 1 1 _]

Step 1: At leftmost 0, replace with X, move right to find a 1
        [X 0 1 1 _]
            →→→
Step 2: Replace that 1 with X, move left back to find next 0
        [X 0 1 X _]
          ←←←
Step 3: Find next 0, replace with X, find next 1...
        [X X 1 X _] → [X X X X _]

Step 4: All crossed off, no unmatched symbols → ACCEPT

If we run out of 1s with 0s remaining (or vice versa) → REJECT
```

## See It Work — TM vs DFA vs PDA

```
Machine    Memory         Languages recognized
───────    ──────         ────────────────────
DFA        None (states)  Regular
PDA        Stack          Context-free
TM         Infinite tape  Recursively enumerable

TM with guaranteed halt → Decidable languages
TM without halt guarantee → Recursively enumerable
```

### Multi-tape and Nondeterministic TMs

- **Multi-tape TM**: Multiple tapes with independent heads. More convenient but NOT more powerful — can be simulated by single-tape TM.
- **Nondeterministic TM**: Can "guess" the right path. Also NOT more powerful — can be simulated by deterministic TM (exponential slowdown).

## How It Actually Works — The Church-Turing Thesis

```
"Any function that can be computed by any mechanical process
 can be computed by a Turing machine."
                                    — Church-Turing Thesis

This is NOT a theorem (can't be proven) — it's a hypothesis.
But no one has ever found a counterexample.

Equivalently powerful models:
  - Lambda calculus (Church, 1936)
  - Turing machines (Turing, 1936)
  - Post correspondence systems
  - Modern programming languages (C, Python, Java)
  - Your laptop (with unlimited memory)
```

### Universal Turing Machine

A single TM that can simulate ANY other TM:
```
Input: ⟨M, w⟩  (description of TM M and input w)
Output: Simulates M on w

This is the theoretical basis of stored-program computers!
Your OS is essentially a universal Turing machine.
```

## Why It Matters

- Turing machines define the boundary between computable and uncomputable.
- The Church-Turing thesis means all programming languages are equivalent in power.
- The universal TM concept led directly to the stored-program computer (von Neumann architecture).
