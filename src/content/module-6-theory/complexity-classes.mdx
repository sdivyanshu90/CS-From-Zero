## Hook — The Biggest Open Problem in Computer Science

Is P = NP? This question asks whether every problem whose solution can be quickly **verified** can also be quickly **solved**. It's worth a $1 million Millennium Prize and affects cryptography, optimization, AI, and virtually every area of CS. Understanding complexity classes tells you which problems are tractable and which are (probably) not.

## P — Polynomial Time

**P** = problems solvable in polynomial time O(nᵏ) by a deterministic TM.

```
Problems in P (efficiently solvable):
  - Sorting: O(n log n)
  - Shortest path: O(V² or (V+E)log V)
  - Primality testing: O(log⁶ n)  — AKS algorithm, 2002
  - Linear programming
  - Matching in bipartite graphs
  - 2-SAT
```

## NP — Nondeterministic Polynomial Time

**NP** = problems whose solutions can be **verified** in polynomial time.

```
Equivalently: solvable in polynomial time by a NONDETERMINISTIC TM
(one that can "guess" the right answer).

Example: SUBSET-SUM
  Input: Set S = \{3, 7, 1, 8, 4\}, target T = 12
  Question: Is there a subset summing to T?
  
  Hard to FIND: Must check many subsets (2ⁿ possibilities)
  Easy to VERIFY: Given \{3, 1, 8\}, check 3+1+8 = 12 ✓
```

**P ⊆ NP**: If you can solve it quickly, you can verify it quickly (just solve it again).

## See It Work — NP-Complete Problems

**NP-Complete** = the hardest problems in NP. If ANY NP-complete problem has a polynomial solution, then P = NP (all of NP collapses to P).

```
Classic NP-Complete problems:
  SAT: Is this boolean formula satisfiable?
  3-SAT: SAT with clauses of size 3
  CLIQUE: Does this graph have a clique of size k?
  VERTEX-COVER: Can k vertices cover all edges?
  HAMILTONIAN-PATH: Visit every vertex exactly once?
  SUBSET-SUM: Does a subset sum to target?
  TRAVELING SALESMAN (decision): Tour of cost ≤ k?
  GRAPH-COLORING: Color with k colors, no adjacent same color?

All NP-complete problems are polynomial-time reducible to each other!
Solving one in polynomial time solves them ALL.
```

### Cook-Levin Theorem (1971)

SAT is NP-complete. Proof: any NP computation can be encoded as a boolean formula.

## How It Actually Works — The Landscape

```
┌──────────────────────────────────────────┐
│                PSPACE                     │
│  ┌────────────────────────────────────┐  │
│  │             NP                      │  │
│  │  ┌──────────────────────────────┐  │  │
│  │  │    NP-Complete              │  │  │
│  │  └──────────────────────────────┘  │  │
│  │  ┌──────────────────────────────┐  │  │
│  │  │         P                    │  │  │
│  │  └──────────────────────────────┘  │  │
│  └────────────────────────────────────┘  │

│  │            co-NP                    │  │
│  └────────────────────────────────────┘  │

              EXPTIME (outside)

P ⊆ NP ⊆ PSPACE ⊆ EXPTIME
Whether any ⊆ is proper (strict subset) is unknown for P vs NP!
```

### NP-Hard

Problems at least as hard as NP-complete, but not necessarily in NP:

```
NP-Hard but NOT in NP:
  - Halting problem (undecidable, not in NP)
  - Optimal Traveling Salesman (optimization, not decision)
  
NP-Complete = NP-Hard ∩ NP
```

## Why It Matters

- If you prove your problem is NP-complete, stop looking for an efficient exact algorithm — use approximation, heuristics, or special cases.
- Cryptography relies on P ≠ NP: if P = NP, RSA and most encryption would break.
- Recognizing NP-complete problems saves enormous time in practice.
