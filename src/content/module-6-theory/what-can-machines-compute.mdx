## Hook — The Limits of Computing

Can a computer solve any problem? Surprisingly, **no**. There are problems that no computer, no matter how powerful, can ever solve. Theory of computation tells us exactly what is and isn't computable — and it was worked out in the 1930s, before computers even existed. Understanding these limits protects you from trying to solve the unsolvable.

## The Big Picture — Chomsky Hierarchy

All computational problems fall into a hierarchy:

```
┌─────────────────────────────────────────────┐
│         Recursively Enumerable              │
│   ┌─────────────────────────────────────┐   │
│   │         Decidable (Recursive)       │   │
│   │   ┌─────────────────────────────┐   │   │
│   │   │     Context-Free            │   │   │
│   │   │   ┌─────────────────────┐   │   │   │
│   │   │   │     Regular         │   │   │   │
│   │   │   └─────────────────────┘   │   │   │
│   │   └─────────────────────────────┘   │   │
│   └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
           UNDECIDABLE (outside all)
```

Each level has more computational power:
- **Regular**: Finite automata (pattern matching)
- **Context-Free**: Pushdown automata (parsing programming languages)
- **Decidable**: Turing machines that always halt (most practical problems)
- **Recursively Enumerable**: Turing machines that may not halt
- **Undecidable**: No machine can solve these at all

## See It Work — Problems at Each Level

```
Level                Machine              Example
─────                ───────              ───────
Regular              Finite automaton     "Does this string match a@b.c?"
Context-Free         Pushdown automaton   "Are parentheses balanced?"
Decidable            Turing machine       "Is this number prime?"
Undecidable          None!                "Does this program halt?"
```

## How It Actually Works — Key Concepts

### Alphabets, Strings, and Languages

```
Alphabet Σ: A finite set of symbols. E.g., Σ = \{0, 1\}
String: A finite sequence over Σ. E.g., "010", "111", ""
Language: A set of strings. E.g., L = \{"0", "01", "001"\}

A computational problem is a language:
  "Is n prime?" = \{strings representing prime numbers\}
  "Does program P halt?" = \{programs that terminate\}

A machine "decides" a language L if:
  Input w ∈ L → machine accepts
  Input w ∉ L → machine rejects
  Machine always halts!
```

### Computation as Recognition

Instead of "computing outputs," theory focuses on **recognizing languages** — accepting or rejecting strings. This captures all computational problems:

```
Decision problem: "Is x in language L?"
  Function problem f(x) → y can be converted to decision:
  "Is (x, y) in the language \{(x,f(x)) | x in domain\}?"
```

## Why It Matters

- Knowing the hierarchy tells you what tools you need (regex vs. parser vs. general algorithm).
- Undecidability means some problems are **provably impossible** — no amount of hardware helps.
- This theory is the mathematical foundation of all of computer science.
