## Hook — Talking Between Processes

Threads share memory, but processes have separate address spaces. How does your web browser talk to the server process? How do microservices communicate? **Inter-Process Communication (IPC)** mechanisms enable processes to exchange data and coordinate — from simple pipes to complex message queues.

## IPC Mechanisms

### Pipes (Unidirectional)

```c
int fd[2];
pipe(fd);  // fd[0]=read end, fd[1]=write end

if (fork() == 0) {
    // Child writes
    close(fd[0]);
    write(fd[1], "hello", 5);
    close(fd[1]);
} else {
    // Parent reads
    close(fd[1]);
    char buf[10];
    read(fd[0], buf, 5);  // buf = "hello"
    close(fd[0]);
}

// Shell pipes: ls | grep ".c" | wc -l
// Each | creates a pipe between processes
```

### Named Pipes (FIFOs)

```bash
mkfifo /tmp/myfifo

# Terminal 1 (writer):
echo "hello" > /tmp/myfifo

# Terminal 2 (reader):
cat /tmp/myfifo  # prints "hello"

# Unlike anonymous pipes, named pipes have a filesystem path
# Unrelated processes can communicate
```

### Shared Memory (Fastest IPC)

```c
// Process A: create and write
int shmid = shmget(IPC_PRIVATE, 4096, IPC_CREAT | 0666);
char *mem = shmat(shmid, NULL, 0);
strcpy(mem, "shared data");

// Process B: attach and read
char *mem = shmat(shmid, NULL, 0);
printf("%s\n", mem);  // "shared data"

// Fastest: no kernel involvement after setup
// But: needs synchronization (semaphores/mutexes)!
```

### Message Queues

```c
// Send
int msqid = msgget(key, IPC_CREAT | 0666);
struct msg { long type; char text[100]; };
struct msg m = {1, "hello"};
msgsnd(msqid, &m, sizeof(m.text), 0);

// Receive
msgrcv(msqid, &m, sizeof(m.text), 1, 0);
// Messages have types — selective receive possible
```

### Sockets (Network IPC)

```
TCP/UDP sockets work between processes on same or different machines.

Local: Unix domain sockets (AF_UNIX) — fast, no network overhead
  /var/run/docker.sock — Docker daemon communication
  /tmp/.X11-unix/X0 — X11 display server

Network: TCP/UDP sockets (AF_INET) — cross-machine
  HTTP, database connections, microservices
```

## How It Actually Works — Comparison

```
Mechanism      Speed       Scope          Complexity
─────────      ─────       ─────          ──────────
Pipe           Medium      Parent-child   Simple
Named Pipe     Medium      Same machine   Simple
Shared Memory  Fastest     Same machine   Needs sync
Message Queue  Medium      Same machine   Medium
Unix Socket    Fast        Same machine   Medium
TCP Socket     Slower      Any machine    Complex
Signal         Fast        Same machine   Very limited (just a number)
```

## Why It Matters

- Unix philosophy: "Do one thing well." IPC connects small programs into powerful pipelines.
- Microservices architecture is entirely built on IPC (typically HTTP/gRPC over sockets).
- Understanding IPC helps you choose the right communication mechanism for your use case.
