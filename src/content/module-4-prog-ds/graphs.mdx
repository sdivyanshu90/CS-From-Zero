## Hook — Connections Are Everything

Facebook's social network. Google Maps' road network. The internet's web of links. Airline routes. Neural networks. All are **graphs** — structures that model **relationships** between things. Unlike trees (which are hierarchical), graphs can have cycles, multiple paths, and complex interconnections. Graphs are arguably the most powerful and general data structure.

## What Is a Graph?

A **graph** G = (V, E) consists of:
- **V**: A set of **vertices** (nodes)
- **E**: A set of **edges** (connections between vertices)

```
Undirected Graph:        Directed Graph:
  A --- B                  A → B
  |   / |                  ↓   ↑
  |  /  |                  C → D
  C --- D

Weighted Graph:
  A --5-- B
  |       |
  3       2
  |       |
  C --7-- D
```

### Graph Types

```
Type          Description                     Example
────          ───────────                     ───────
Undirected    Edges go both ways              Facebook friendships
Directed      Edges have direction            Twitter follows, web links
Weighted      Edges have costs/distances      Road map with distances
Unweighted    All edges equal                 Social network connections
Cyclic        Contains cycles                 Road networks
Acyclic       No cycles                       Family tree
DAG           Directed Acyclic Graph          Task dependencies, Git commits
```

## See It Work — Graph Representations

### Adjacency Matrix

```
     A  B  C  D
A  [ 0  1  1  0 ]
B  [ 1  0  1  1 ]
C  [ 1  1  0  1 ]
D  [ 0  1  1  0 ]

matrix[i][j] = 1 if edge exists between i and j

Space: O(V²)
Check edge exists: O(1) — just look up matrix[i][j]
Find all neighbors: O(V) — scan entire row
Good for: Dense graphs (many edges)
```

### Adjacency List

```
A: [B, C]
B: [A, C, D]
C: [A, B, D]
D: [B, C]

Each vertex stores a list of its neighbors.

Space: O(V + E)
Check edge exists: O(degree) — scan neighbor list
Find all neighbors: O(degree) — just return the list
Good for: Sparse graphs (few edges relative to V²)
```

Most real-world graphs are sparse, so adjacency lists are the standard representation.

## How It Actually Works — Graph Traversals

### BFS (Breadth-First Search) — Level by Level

```
From A:
Queue: [A]
Visit A, enqueue neighbors B,C → Queue: [B, C]
Visit B, enqueue unvisited D   → Queue: [C, D]
Visit C                        → Queue: [D]
Visit D                        → Queue: []

Order: A, B, C, D
Time: O(V + E)

BFS finds shortest paths in unweighted graphs!
```

### DFS (Depth-First Search) — Go Deep First

```
From A:
Stack: [A]
Visit A, push C,B → Stack: [C, B]
Visit B, push D   → Stack: [C, D]
Visit D, push C (already visited) → Stack: [C]
Visit C           → Stack: []

Order: A, B, D, C
Time: O(V + E)

DFS is useful for: cycle detection, topological sort, connected components
```

### Topological Sort (DAGs only)

Order vertices so that every edge goes from earlier to later:

```
Task dependencies:
  A → C (A must finish before C)
  B → C
  B → D
  C → E
  D → E

Topological order: A, B, C, D, E (or A, B, D, C, E)
Both valid — every edge goes left to right.

Used for: Build systems (Makefile), course prerequisites, task scheduling.
```

## Why It Matters

- Graphs model **relationships** — the most general type of data structure.
- Social networks, maps, internet routing, recommendation systems, and compilers all use graphs.
- Graph algorithms (BFS, DFS, shortest path, minimum spanning tree) solve real-world problems.
- Understanding graphs is prerequisite to understanding many advanced CS topics.
