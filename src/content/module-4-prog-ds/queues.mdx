## Hook — First Come, First Served

When you wait in line at a coffee shop, the first person in line gets served first. That's a **queue** — a **First-In, First-Out (FIFO)** data structure. Queues are everywhere in computing: print job scheduling, web server request handling, breadth-first search, and message passing between systems.

## What Is a Queue?

A **queue** supports two primary operations:

- **enqueue(item)**: Add an item to the back (rear).
- **dequeue()**: Remove and return the item from the front.
- **peek()** / **front()**: Look at the front item without removing it.
- **isEmpty()**: Check if the queue is empty.

```
enqueue(10)  enqueue(20)  enqueue(30)  dequeue()  dequeue()

 Front→ │10│   │10│20│   │10│20│30│   │20│30│    │30│
         ←Rear                          → 10      → 20
```

**All operations are O(1).**

## See It Work — Implementing a Queue

### Circular Array Queue

A simple array queue wastes space as the front moves forward. A **circular array** wraps around:

```c
#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int front, rear, size;
} Queue;

void init(Queue *q) { q->front = 0; q->rear = -1; q->size = 0; }
int isEmpty(Queue *q) { return q->size == 0; }
int isFull(Queue *q) { return q->size == MAX_SIZE; }

void enqueue(Queue *q, int value) {
    if (isFull(q)) { printf("Queue full!\n"); return; }
    q->rear = (q->rear + 1) % MAX_SIZE;  // wrap around!
    q->data[q->rear] = value;
    q->size++;
}

int dequeue(Queue *q) {
    if (isEmpty(q)) { printf("Queue empty!\n"); return -1; }
    int value = q->data[q->front];
    q->front = (q->front + 1) % MAX_SIZE;  // wrap around!
    q->size--;
    return value;
}
```

```
Circular array visualization (capacity 5):
                     front
                       ↓
          [30][40][ ][ ][20]
                 ↑
                rear

After enqueue(50): front=4, rear=2
          [30][40][50][ ][20]
```

### Queue Variants

**Double-Ended Queue (Deque)**: Insert and remove from BOTH ends.

```
addFront() ←  [10][20][30]  → addRear()
removeFront() ←             → removeRear()
```

**Priority Queue**: Elements have priorities; highest priority is dequeued first (regardless of insertion order). Usually implemented with a heap.

```
enqueue(task_A, priority=3)
enqueue(task_B, priority=1)
enqueue(task_C, priority=5)
dequeue() → task_C (priority 5, highest)
dequeue() → task_A (priority 3)
dequeue() → task_B (priority 1)
```

## How It Actually Works — BFS with a Queue

**Breadth-First Search** uses a queue to explore level by level:

```
Graph:  A → B, C
        B → D
        C → E

BFS with queue:
  Enqueue A         Queue: [A]
  Dequeue A, visit   Queue: []  Enqueue neighbors B, C
  Dequeue B, visit   Queue: [C]  Enqueue neighbor D
  Dequeue C, visit   Queue: [D]  Enqueue neighbor E
  Dequeue D, visit   Queue: [E]  No unvisited neighbors
  Dequeue E, visit   Queue: []
  
  Visit order: A, B, C, D, E (breadth-first — level by level!)
```

Compare with DFS (stack): A, B, D, C, E — depth first.

### Real-World Queue Applications

```
Application              What's Queued          Why FIFO
──────────               ─────────              ────────
Print spooler            Print jobs             Fair: first submitted prints first
Web server               HTTP requests          Fair: first request served first
CPU scheduling           Processes              Round-robin time sharing
Message queues           Messages (Kafka, RabbitMQ) Ordered delivery
Key buffer               Keystrokes             Type order preserved
Network router           Packets                First-come first-forwarded
```

## Why It Matters

- Queues ensure **fairness** — first come, first served.
- BFS requires a queue and finds **shortest paths** in unweighted graphs.
- Operating systems use queues for process scheduling, I/O request ordering, and inter-process communication.
- Message queues (Kafka, RabbitMQ, SQS) are fundamental to modern distributed systems.
