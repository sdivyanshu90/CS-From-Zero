## Hook — Data as Mathematics

In 1970, Edgar Codd proposed organizing data as **relations** — mathematical sets of tuples. This sounds abstract, but it's simply: tables with rows and columns. The relational model gave us a rigorous foundation for querying data, and it powers virtually every database you use today.

## Terminology

```
Relational Term    ↔    Informal Term
───────────────         ──────────────
Relation               Table
Tuple                  Row / Record
Attribute              Column / Field
Domain                 Data type (INT, VARCHAR, ...)
Degree                 Number of columns
Cardinality            Number of rows
Schema                 Table structure (name + attributes)
Instance               Current data in the table
```

## See It Work — Relational Algebra

Relational algebra is the formal query language — SQL is built on it.

```
Six fundamental operations:

1. Selection (σ): Filter rows
   σ(gpa > 3.5)(Student) → students with gpa > 3.5
   SQL: SELECT * FROM Student WHERE gpa > 3.5

2. Projection (π): Choose columns
   π(name, gpa)(Student) → only name and gpa columns
   SQL: SELECT name, gpa FROM Student

3. Union (∪): Combine rows from two compatible relations
   R ∪ S → all rows in R or S (no duplicates)
   SQL: SELECT * FROM R UNION SELECT * FROM S

4. Set Difference (−): Rows in R but not in S
   R − S
   SQL: SELECT * FROM R EXCEPT SELECT * FROM S

5. Cartesian Product (×): Every row of R paired with every row of S
   R × S → |R| × |S| rows
   SQL: SELECT * FROM R, S  (rarely useful alone)

6. Rename (ρ): Rename a relation or attributes
   ρ(NewName)(R)
   SQL: SELECT name AS student_name FROM R
```

### Joins (Derived from Cartesian Product + Selection)

```
Natural Join (⋈): Match rows on common attribute names
  Student ⋈ Enrollment
  → pairs where Student.id = Enrollment.student_id
  SQL: SELECT * FROM Student NATURAL JOIN Enrollment

Theta Join: Cartesian product + arbitrary condition
  R ⋈(R.a > S.b) S
  SQL: SELECT * FROM R JOIN S ON R.a > S.b

Equi-Join: Theta join where condition is equality
  SQL: SELECT * FROM R JOIN S ON R.id = S.id
```

## How It Actually Works — Integrity Constraints

```
Entity Integrity:
  Primary key values must be unique and NOT NULL.
  Every row must be identifiable.

Referential Integrity:
  Foreign key values must reference existing PK values
  (or be NULL).
  DELETE student → what about their enrollments?
    CASCADE:    Delete enrollments too
    SET NULL:   Set enrollment.student_id = NULL
    RESTRICT:   Prevent deletion if enrollments exist

Domain Constraints:
  Values must be of the correct type and range.
  age INT CHECK (age >= 0 AND age <= 150)
```

## Why It Matters

- The relational model's mathematical foundation is why SQL works — every SQL query maps to relational algebra operations.
- Understanding relational algebra helps you write better queries and understand query optimization.
- Integrity constraints prevent bad data from entering your database — enforced automatically.
